================================================================================
                    COMMIT MESSAGE CONVENTION
================================================================================

Format:
  type(scope): subject
  
  body
  
  footer

Types:
  • feat: New feature
  • fix: Bug fix
  • docs: Documentation
  • style: Code style
  • refactor: Code refactoring
  • test: Testing
  • chore: Build/dependencies

Example:
  feat(event-system): Add EventBridge integration
  
  - Implement EventBridge configuration
  - Create event routing rules
  - Set up SQS queues
  
  Closes #123

================================================================================
                    SECURITY BEST PRACTICES
================================================================================

Never Commit:
  X Passwords or API keys
  X Private credentials
  X Environment-specific files
  X Large binary files
  X Build artifacts
  X IDE configuration

Use Instead:
  ✓ Environment variables
  ✓ .env.example files
  ✓ Configuration templates
  ✓ Secrets Manager
  ✓ GitHub Secrets (for CI/CD)

Ignored Files:
  • IDE files (.idea/, .vscode/)
  • Build artifacts (target/, dist/)
  • Dependencies (node_modules/)
  • Environment files (.env)
  • Logs (*.log)
  • OS files (.DS_Store)
  • Secrets (secrets.json)
